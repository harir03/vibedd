# 15 — Vector Memory (ChromaDB)

**Tier:** 3 (Secondary WOW)  
**Status:** IMPLEMENTED  
**Priority:** 3.4

## What This Does

A **vector database** that stores embeddings (numerical representations) of past incidents, allowing the system to find **similar past events** instantly. When a new alert comes in, we search the vector store for "what incidents looked like this before?" This is the AI's **long-term memory**.

### Why It Matters
Without vector memory, the system starts from zero every time. With it:
- "This attack looks 94% similar to the DDoS on January 15th"
- Predictive analysis (Feature 13) can forecast next attack stages
- Playbook generation (Feature 09) can suggest responses that worked before
- Self-evolving agent (Feature 10) remembers what patterns it already learned

### What to Build

#### 1. ChromaDB Integration
```python
import chromadb

client = chromadb.PersistentClient(path="/data/chromadb")
collection = client.get_or_create_collection(
    name="incidents",
    metadata={"hnsw:space": "cosine"}  # Cosine similarity
)

# Store an incident's embedding
collection.add(
    ids=[incident.id],
    embeddings=[incident.embedding],           # Vector from Ollama's embed API
    metadatas=[{
        "category": incident.category,
        "severity": incident.severity,
        "source_ips": json.dumps(incident.source_ips),
        "timestamp": incident.timestamp.isoformat(),
    }],
    documents=[incident.summary]              # Text summary for hybrid search
)

# Find similar incidents
results = collection.query(
    query_embeddings=[new_alert.embedding],
    n_results=5,
    where={"severity": {"$in": ["high", "critical"]}}
)
```

#### 2. Embedding Generation
Use Ollama's embedding API to convert text → vectors:
```typescript
// Node.js side
const embedding = await ollama.embeddings({
  model: 'mistral',
  prompt: `${alert.category}: ${alert.method} ${alert.path} from ${alert.sourceIP} - ${alert.aiAnalysis}`
});
// embedding.embedding = [0.123, -0.456, ...] (4096-dim vector)
```

#### 3. Collections to Maintain
```
incidents        — full incident summaries with metadata
alert_patterns   — individual alert patterns for similarity matching
attack_payloads  — known malicious payloads for pattern recognition
learned_rules    — rules generated by the self-evolving agent
playbooks        — generated playbooks for similar incident lookup
```

### Architecture
```
New Alert → Generate Embedding (Ollama) → Query ChromaDB for similar past incidents
                                        → Enrich alert with historical context
                                        → Feed to Features 07, 08, 09, 13

Confirmed Incident → Generate Embedding → Store in ChromaDB for future reference
```

### Tech Stack
- **ChromaDB** — vector database (runs as Docker container)
- **Ollama** — embedding generation (uses same instance as threat analysis)
- **Python** — ChromaDB client (or Node.js via chromadb npm package)
